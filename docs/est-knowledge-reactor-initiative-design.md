# Knowledge Reactor: Initiative Design Blueprint

## Purpose

This document applies the EST game-arc rubric to the first Knowledge Reactor strand:

- `Initiative`

It translates the source material into:
- player fantasy
- core verb
- gameplay loop
- progression arc
- ecosystem rewards
- content structures we can build in GitHub

This is the first pass for turning the Knowledge Reactor into a real game space instead of a revision panel.

---

## Source Content Summary

From the Initiative sheet, the core curriculum ideas are:

- the concept of initiative
- five ways to show initiative at work:
  - being proactive
  - improving work practices
  - vocalising opinions
  - helping fellow workers
  - seeking more responsibilities
- why employers value initiative
- examples from:
  - new workers
  - experienced workers / supervisors
- how initiative supports teamwork
- personal traits that support initiative:
  - confidence
  - motivation
  - curiosity
  - empathy
  - willingness to learn

The sheet also gives us:
- retail and workplace examples
- action-rich scenarios
- short-answer EST prompt directions

This means initiative is not a "definition only" topic.
It is really a:
- judgement topic
- workplace behaviour topic
- teamwork-and-decision topic

That makes it ideal for gameplay.

## Initiative Through The New Lens

## 1. Player Fantasy

The student should feel like:

- they are a worker entering a live workplace
- they must spot opportunities, problems, and team needs
- they must decide how to act before being told

The fantasy is:

- `I am proving I can act like an enterprising worker`

Not:

- `I am reading statements about initiative`

---

## 2. Core Player Verb

The core verb for Initiative should be:

- `spot`
- `decide`
- `step in`
- `improve`

That means the player should mostly be doing things like:

- identifying a workplace moment
- choosing the strongest next move
- spotting whether a worker showed initiative or missed it
- deciding what kind of initiative is being demonstrated

---

## 3. Why Initiative Fits Knowledge Reactor

Knowledge Reactor should be where students:
- learn the content
- recognise patterns
- make quick content-based decisions

Initiative fits this perfectly because students need to learn:
- what initiative is
- what it looks like in context
- the different forms it can take
- how it overlaps with teamwork and communication

So Initiative inside Knowledge Reactor should feel like:

- `behaviour reactor`
- `workplace judgement chamber`
- `initiative scanner`

---

## Recommended Initiative Arc

The Initiative strand should follow this progression:

1. `Recognise initiative`
2. `Classify the type of initiative`
3. `Distinguish strong initiative from weak action`
4. `Connect initiative to teamwork and worker traits`
5. `Use that thinking in EST-style explanation`

That gives a real mastery curve instead of one repeated mechanic.

---

## Recommended Game Structure

I recommend Initiative in Knowledge Reactor becomes a short 4-part sequence.

This sequence should quietly support the five initiative types as a light memory aid:

- `P` = Proactive
- `I` = Improve work practices
- `V` = Vocalise opinions
- `H` = Help fellow workers
- `S` = Seek more responsibilities

This can appear as:
- a small support ribbon
- a recap badge row
- a low-key visual anchor inside the strand

It should not become:
- the main theme
- the main branding
- a forced chant or gimmick

The real focus should stay on:
- classification
- example -> label -> justification
- sentence construction

### Phase 1: Spot The Move

Purpose:
- teach the player to recognise whether initiative is happening

Best mechanic:
- `scenario`
- `detective-lite`

Player action:
- a workplace moment appears
- player decides whether the worker is:
  - showing initiative
  - missing initiative
  - only partially showing initiative

Why it works:
- immediate judgement
- low friction
- strong link to real work behaviour

This trains:
- `classification`

Example prompt:
- A worker notices shelves are nearly empty and restocks them before customers complain.

Player chooses:
- Strong initiative
- Not initiative
- Needs manager instruction first

### Phase 2: Name The Type

Purpose:
- teach the five ways workers can show initiative

Best mechanic:
- `sort`
- `classification`

Player action:
- player classifies workplace actions into:
  - proactive
  - improving work practices
  - vocalising opinions
  - helping fellow workers
  - seeking more responsibilities

Why it works:
- makes the five categories visible
- supports fast recall later in the EST

This trains:
- `example -> label`

Example:
- "Mentors a new worker during a busy shift"

Answer:
- Helping fellow workers

### Phase 3: New Worker vs Supervisor

Purpose:
- show how initiative looks different at different experience levels

Best mechanic:
- `compare`
- `scenario`

Player action:
- compare a new worker response with an experienced worker response
- choose which one best fits the situation
- or identify how initiative changes with responsibility level

Why it works:
- uses the source sheet directly
- helps students see that initiative is scalable, not one-size-fits-all

This phase can also absorb the strongest part of the "fix the workplace" idea:
- a workplace problem appears
- the player chooses which response shows the strongest initiative
- the feedback explains why that move is stronger

This trains:
- `label -> justification`

Example:
- New worker:
  - report low stock
- Supervisor:
  - redesign staff allocation before queues grow

### Phase 4: Team Pulse

Purpose:
- connect initiative to teamwork, communication, and workplace traits

Best mechanic:
- `scenario`
- `builder-lite`

Player action:
- choose which trait or team effect best matches the initiative moment

Possible targets:
- confidence
- motivation
- curiosity
- empathy
- willingness to learn
- teamwork support
- safety support

Why it works:
- deepens the concept
- stops initiative from being reduced to "working alone"

This phase should end in a short EST-style writing move:
- improve a weak sentence
- or complete a scaffold such as:
  - `I demonstrated initiative by ...`
  - `This shows initiative because ...`
  - `This improved the workplace/team because ...`

This trains:
- `sentence construction`

So in practice, the four-part Initiative sequence becomes:

1. `Spot the Initiative`
2. `Name the Initiative Type`
3. `Fix the Workplace`
4. `Upgrade the Answer`

## Recommended Initiative Hero Loop

If Initiative gets one signature mechanic inside Knowledge Reactor, it should be:

- `Workplace Signal Scanner`

### The loop

1. A workplace moment appears
2. The player identifies whether initiative is present
3. The player classifies the initiative type
4. The player sees the workplace/team result
5. The player banks reactor energy / readiness / salary

This loop matches the curriculum very well because the student is doing:
- recognition
- classification
- consequence thinking

That is much more game-like than reading a paragraph and clicking next.

The full learning ladder inside the loop should be:

- example
- label
- justification
- sentence

That is what makes it much closer to EST performance prep than plain quiz revision.

## What Initiative Should Feel Like

### Good version

It should feel like:
- scanning the workplace for smart moves
- noticing the best next action
- stepping in to help the team succeed
- predicting what helps a workplace run better

### Bad version

It should not feel like:
- matching long definitions to labels
- reading blocks of content and answering isolated multiple-choice questions
- repeating the same mechanic without any shift in difficulty or role

## Narrative Wrapper For Initiative

This strand should be framed as:

- `Initiative Reactor`
- `Workplace Signal Chamber`
- `Action before instruction`

Suggested mission premise:

- The workplace system is unstable.
- Customers, staff, safety, and workflow are all sending signals.
- Your job is to detect the strongest initiative move before the problem grows.

This gives:
- urgency
- purpose
- a reason for rapid decision-based play

## How Initiative Links To EST Goals

Initiative should support the final EST by training students to:

- define initiative clearly
- identify examples of initiative in context
- distinguish different forms of initiative
- connect initiative to teamwork and worker traits
- use specific examples in a short-answer response

This strand directly supports questions such as:

- What is initiative?
- Describe ways initiative can be shown at work.
- Explain examples of initiative.
- Show how initiative links with teamwork or employability development.

So Initiative in Knowledge Reactor should not try to become the final writing task.
Its job is:

- `recognition`
- `classification`
- `applied explanation prep`

The actual longer response can then happen later in the simulation or short written note.

## Initiative Against The Rubric

### Agency

Good Initiative design should let the player:
- choose the strongest move
- classify behaviour
- compare worker responses

This is strong for agency.

### Safe Failure

Wrong choices should:
- explain why the action is weaker
- show what a stronger initiative move would look like
- keep the player moving

No silent dead ends.

### Narrative Purpose

Initiative must feel like:
- stabilising a workplace through better worker behaviour

### Progression

Progression should move from:
- recognition
- to classification
- to comparison
- to explanation readiness

### Friction Removal

Keep each screen focused on:
- one scenario
- one judgement
- one visible consequence

### Intrinsic Engagement

Even without points, the player should enjoy:
- spotting the smart move
- seeing the workplace consequence
- feeling clever for reading the moment correctly

### Ecosystem Link

Initiative should visibly feed:
- salary
- readiness
- class contribution
- maybe strongest-skill identity

## Recommended First Build For Initiative

If we keep this manageable, the first good version should contain:

1. `Signal Scan`
- identify whether a workplace action shows initiative

2. `Initiative Sort`
- classify into one of the five initiative types

3. `Fix the Workplace`
- choose the strongest initiative response in a workplace scenario
- use new worker vs supervisor examples where useful

4. `Upgrade the Answer`
- improve a weak EST-style sentence or complete a scaffolded initiative response

That is enough for a strong first-pass Initiative module.

It gives:
- variety
- content accuracy
- gameplay progression
- direct EST usefulness

without trying to overbuild.

---

## Build Guardrails

When building Initiative, avoid:

- long text walls on screen
- using only multiple-choice the whole way through
- turning all five initiative types into flat memorisation cards
- forcing writing too early
- separating initiative from teamwork completely

The strongest version will keep:
- context
- action
- consequence
- role difference
- team connection
