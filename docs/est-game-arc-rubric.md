# EST Game Arc And Design Rubric

## Purpose

This document reframes the EST Revision module as a game system rather than a collection of revision pages.

The goal is to make sure all four EST labs:
- serve one larger EST goal
- feel like parts of one coherent game arc
- feed the wider Career Empire ecosystem
- avoid sliding back into "poshed-up worksheet" design

This is the design checkpoint we should use before building more EST content.

---

## The Core EST Fantasy

The EST module should feel like:

- a `high-stakes exam prep simulator`
- a `training facility` where students sharpen different exam abilities
- a `career launch system` where mastery turns into marks, readiness, salary, tax contribution, class impact, and status

The player fantasy is not:
- "I am filling out revision activities"

The player fantasy is:
- "I am upgrading into an EST-ready operator"

That means each lab should answer a different player need:

- `Knowledge Reactor`: Do I know the content?
- `Glossary Check`: Can I use exact EST language under pressure?
- `VTCS Decoder`: Can I read the question properly before I answer?
- `Boss Round`: Can I perform in the final simulation and produce a mark-worthy response?

---

## The Overall EST Arc

The EST module should follow this arc:

1. `Stabilise knowledge`
2. `Sharpen language`
3. `Decode the challenge`
4. `Perform under pressure`
5. `Bank career rewards`

This gives the player a real progression curve:

- first they gather control
- then they gain precision
- then they gain strategic thinking
- then they prove performance
- then they convert that into visible life/career rewards

That is a much stronger structure than four unrelated mini-activities.

---

## The Four-Lab Journey

### 1. Knowledge Reactor

Primary role:
- build curriculum knowledge and applied topic judgement

Game purpose:
- this is the `content engine`
- it should feel like activating or stabilising the main knowledge systems required for the EST

Player verb:
- choose
- sort
- judge
- respond

What it should train:
- topic familiarity
- confidence with scenarios
- quick content recognition
- the ability to explain content in short EST-style language

Why it matters to the bigger arc:
- this is where students assemble the raw material they will later need in Glossary, Decoder, and Boss Round

### 2. Glossary Check

Primary role:
- convert vague knowledge into exact language

Game purpose:
- this is the `precision engine`
- it should feel like repairing terminology, restoring signal accuracy, or locking in high-value language

Player verb:
- recover
- discriminate
- retrieve
- lock in

What it should train:
- term-definition precision
- retrieval fluency
- discrimination between similar terms
- exam language confidence

Why it matters to the bigger arc:
- this takes content from "kind of know it" to "can use it properly"

### 3. VTCS Decoder

Primary role:
- teach students how to read the question before they answer it

Game purpose:
- this is the `forensics engine`
- it should feel like cracking the code of the question so the player knows what the examiner is really asking

Player verb:
- inspect
- decode
- classify
- frame

What it should train:
- command verb recognition
- topic identification
- context recognition
- response structure selection

Why it matters to the bigger arc:
- this is the bridge between knowing things and answering well

### 4. Boss Round

Primary role:
- simulate EST performance under pressure

Game purpose:
- this is the `performance engine`
- it should feel like the culmination of all previous training, not just another page with a textbox

Player verb:
- build
- justify
- execute
- submit

What it should train:
- answer construction
- explanation quality
- glossary integration
- strategic structure use

Why it matters to the bigger arc:
- this is where the module proves whether all earlier labs actually prepared the student

---

## How The Four Labs Link Together

Each lab should feed the next in a visible way.

### The intended chain

- `Knowledge Reactor` feeds content confidence
- `Glossary Check` feeds language precision
- `VTCS Decoder` feeds question strategy
- `Boss Round` consumes all three and tests final performance

### Player-facing interpretation

- "I know the ideas."
- "I can name them accurately."
- "I can decode what the question wants."
- "I can now produce the answer."

If a lab can be removed without changing that chain, it is probably not essential enough or not designed clearly enough.

## EST Rubric: Game Or Worksheet?

This is the build rubric we should use on every EST lab and mechanic.

### 1. Agency

Ask:
- Does the player make meaningful choices?
- Is there more than one path, tactic, or response pattern?
- Does the student feel like they are doing something, not just reacting to prompts?

If no:
- it is drifting toward worksheet mode

### 2. Safe Failure

Ask:
- Does a wrong move teach the student what to do differently?
- Is failure recoverable and non-shaming?
- Can they try again without the whole experience feeling punitive?

If no:
- it will feel like assessment disguised as gameplay

### 3. Narrative Purpose

Ask:
- Does the activity have a role inside the EST mission?
- Does the student know why this lab exists?
- Does it feel like part of a larger campaign?

If no:
- it will feel like isolated task blocks

### 4. Progression

Ask:
- Does the player feel they are moving through a journey?
- Is new pressure, difficulty, or responsibility introduced over time?
- Can they see mastery growing?

If no:
- it will feel flat even if it looks polished

### 5. Friction Removal

Ask:
- Are the instructions clear?
- Is the screen focused on one meaningful action at a time?
- Is there any dead clicking, hidden rule, or silent failure state?

If no:
- it will feel like a bad digital worksheet

### 6. Intrinsic Engagement

Ask:
- If we removed salary, points, or badges, would this still be satisfying to play?
- Does the interaction itself feel interesting?

If no:
- rewards are masking a weak mechanic

### 7. Ecosystem Link

Ask:
- Does the player understand how this lab affects their bigger Career Empire?
- Does it visibly connect to salary, tax, community, leaderboard, shop, or profile growth?

If no:
- the module risks feeling disconnected from the platform

## Ecosystem Mapping

The EST module should contribute to the wider game in a very clear way.

### Salary

Meaning:
- salary represents the career value of strong EST preparation

How labs should feed it:
- better performance
- improved best score
- consistent mastery
- final simulation success

### Tax / Community

Meaning:
- part of the player’s success contributes to something larger than them

How labs should feed it:
- each lab should generate a visible class/community tax contribution
- some labs can ask the player where they want that contribution directed

### Leaderboard

Meaning:
- the leaderboard should reflect strong play, not shallow grinding

How labs should feed it:
- mastery percentage
- completion quality
- best score improvements
- possibly readiness milestones

### Shop

Meaning:
- shop rewards make academic improvement feel tied to lifestyle progression

How labs should feed it:
- earnings convert into spendable progress
- but only through meaningful performance, not repetitive farming

### Student Hub

Meaning:
- the hub should explain the player’s journey back to them

How labs should feed it:
- show earnings log
- show best scores
- show progression through the four EST labs
- show readiness growth and current weak area

## Design Role Of Each Lab In The Ecosystem

### Knowledge Reactor

Ecosystem role:
- the broadest content contributor
- likely the best place to drive readiness and content mastery

Main reward identity:
- `knowledge income`

### Glossary Check

Ecosystem role:
- best precision multiplier
- should improve quality rather than just volume

Main reward identity:
- `precision income`

### VTCS Decoder

Ecosystem role:
- strategic multiplier
- should feel like it improves answer efficiency and protects marks

Main reward identity:
- `strategy income`

### Boss Round

Ecosystem role:
- highest-stakes proof of mastery
- should be the strongest reward pulse because it uses all earlier learning

Main reward identity:
- `performance income`

---

## What "Good" Looks Like For Each Lab

### Knowledge Reactor should feel like:
- activating content systems
- clearing revision sectors
- stabilising knowledge under light pressure

It should not feel like:
- a list of topic quizzes

### Glossary Check should feel like:
- restoring language accuracy
- distinguishing close concepts
- becoming sharper and faster

It should not feel like:
- a match-the-definition worksheet with fancy borders

### VTCS Decoder should feel like:
- cracking the examiner’s code
- detecting hidden demands
- building tactical advantage

It should not feel like:
- selecting four labels from a form

### Boss Round should feel like:
- the actual test moment
- a real submission
- the payoff for the whole training run

It should not feel like:
- one last textbox after some mini-games

## Current Strengths In The EST Module

Based on the live structure, these are already working in the right direction:

- there is a clear four-stage structure
- the stage naming already suggests distinct functions
- rewards already connect into salary, tax, readiness, and marks
- the module already saves progress and writes into the wider ecosystem
- the glossary now has more of a game shell than before

These are strong foundations.

---

## Current Risks

These are the main design risks to keep watching:

- labs can still become too panel-heavy
- some interactions still risk feeling like "worksheet but clickier"
- if instructions are unclear, the game fantasy collapses immediately
- if best-score logic is not respected, replay becomes farming
- if stage-to-stage links are invisible, the four labs can feel disconnected

---

## The Next Design Target

The next design goal should be:

- make each lab feel like a different kind of play
- while keeping the same larger EST mission

That means:

- `Knowledge Reactor` should become the strongest content-judgement space
- `Glossary Check` should stay as the precision-retrieval space
- `VTCS Decoder` should become the tactical question-forensics space
- `Boss Round` should feel like the final exam simulation space

Each one should have:
- its own fantasy
- its own immediate player verb
- its own style of feedback
- its own reward identity

But all four should clearly feed:
- EST mastery
- readiness
- salary
- tax/community
- leaderboard
- shop

---

## Build Rule For Next Passes

Before we build or revise any EST lab, we should answer:

1. What is the player fantasy here?
2. What is the core verb?
3. What makes this different from the other three labs?
4. How does failure teach instead of punish?
5. What gets rewarded in the wider ecosystem?
6. Would this still be engaging without points?

If we cannot answer those clearly, the mechanic is not ready yet.

---

## Immediate Follow-On Work

The next practical step is:

- use this document to redesign `Knowledge Reactor` first

Why:
- it is the largest content surface
- it sets the tone for EST as a whole
- if it feels strong, the whole module feels stronger
- it is the best place to make the overall EST arc feel real rather than decorative

After that:

- revisit `VTCS Decoder`
- then strengthen `Boss Round`
- keep `Glossary Check` stable unless a specific bug or friction point appears
