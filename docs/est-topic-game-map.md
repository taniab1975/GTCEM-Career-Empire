# EST Topic To Game Map

## Purpose

This map pairs each EST revision topic with the game style that best fits the kind of thinking students need to practise.

The goal is not to force every topic into the same mechanic. The goal is to match:
- recall content with recall-heavy games
- judgement content with scenario games
- structure content with answer-building games
- interpretation content with evidence and detective games

This should help the EST module feel varied, visual, and purposeful inside the main GitHub-built game.

## Design Rule

- Use `quiz`, `flashcards`, and rapid match tasks for definitions, glossary, and direct recall.
- Use `scenario`, `roleplay`, and `decision` games for behavioural or workplace judgement content.
- Use `builder`, `sort`, and `sequence` games for answer structure and applied technique.
- Use `detective`, `evidence scan`, and `compare` games for spotting problems, judging quality, and interpreting information.

| Short-answer structure | Build-the-answer, response skeleton, marking-key challenge | This is about constructing mark-worthy answers, not just knowing content | Sentence-block builders, mark ladders, examiner comments, missing-step highlights | Build the strongest answer, compare two responses, add what is needed for full marks |

## Recommended Core Game Families

These are the main game families the EST module should keep reusing.

### 1. Recall Games

Best for:
- glossary
- command verbs
- direct content definitions

Examples:
- flashcards
- quick quiz
- match term to definition
- choose the strongest answer point

### 2. Scenario Games

Best for:
- initiative
- communication
- personal finance
- unexpected life events
- future of work applications

Examples:
- what should happen next
- choose the best response
- roleplay conversation
- consequence-based branching choice

### 3. Builder Games

Best for:
- STAR
- selection criteria
- cover letters
- short-answer structure
- command verb to answer pattern

Examples:
- order the response
- drag parts into the strongest answer
- fix a weak answer
- add the missing evidence or explanation

### 4. Detective Games

Best for:
- workplace communication scenes
- initiative scenes
- labour market information
- identifying weak job applications
- spotting missing parts in an answer

Examples:
- find what is wrong
- choose what matters most
- match evidence to claim
- identify what would cost marks

## Visual Direction For The GitHub Build

Even though this is being built in the main game rather than Canva, it still makes sense to borrow the strongest visual ideas:

- Use scene-based cards instead of plain question lists where the topic is behavioural.
- Use visible boards, panels, and meters for planning and finance topics.
- Use drag/drop or tap-to-sort patterns for structure-heavy content.
- Use strong icon systems and colour coding so students can recognise patterns fast.
- Use annotated weak-vs-strong examples for job application and EST writing tasks.
- Use one visual language per game family so the module feels cohesive but not repetitive.

## Practical Build Recommendation

If we want the EST module to feel polished without exploding the workload, use a small set of reusable mechanics:

- `quiz`
- `flashcards`
- `sort`
- `scenario`
- `builder`
- `detective`
- `compare`

Then assign them by topic rather than inventing a brand-new mechanic every time.

That gives:
- variety for students
- cleaner code reuse
- faster content production
- stronger visual identity across the module
